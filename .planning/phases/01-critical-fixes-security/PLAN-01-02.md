# Plan 01-02: Security Audit - Env Files, Secrets, CSP Hardening

**Phase:** 1 - Critical Fixes & Security
**Requirements:** FIX-03, SEC-01, SEC-03
**Status:** Pending

## Goal
Ensure no secrets are committed to the repository, tighten CSP for production, and audit all env files.

## Context
- `apps/frontend/.env.production` is tracked by git and contains what appear to be real values for NEXTAUTH_SECRET, SUPABASE_ANON_KEY, DATABASE_URL
- `.gitignore` already covers `.env` but NOT `.env.production` specifically
- CSP in `apps/frontend/next.config.ts` has `connect-src` allowing `http://local-backend http://localhost:*` in production

## Tasks

### Task 1: Audit .env.production for real secrets
Read `apps/frontend/.env.production` and classify each value:
- `NEXTAUTH_SECRET` -- if this is a real production secret, it MUST be removed from git
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` -- this is a public anon key (safe to commit, it's designed to be public)
- `NEXT_PUBLIC_SUPABASE_URL` -- public URL (safe)
- `DATABASE_URL` -- contains placeholder `postgresql://placeholder:***@localhost/placeholder` (safe)
- `NEXTAUTH_URL` -- contains URL only (safe if no secret embedded)
- `NEXT_PUBLIC_URL`, `NEXT_PUBLIC_APP_URL`, `NEXT_PUBLIC_BACKEND_URL` -- public URLs (safe)

### Task 2: Remove .env.production from git tracking
```bash
# Remove from git but keep locally
git rm --cached apps/frontend/.env.production
```

Add to `.gitignore`:
```
# Environment files with secrets
.env.production
.env.*.local
```

Create `.env.production.example` with placeholder values (if not already present):
```bash
# Verify example file exists
ls apps/frontend/.env.production.example
```

### Task 3: Rotate exposed secrets
If NEXTAUTH_SECRET was a real production value (not a placeholder):
1. Generate a new secret: `openssl rand -base64 32`
2. Update the production deployment with the new secret
3. The old secret in git history should be noted (consider `git filter-branch` or BFG if critical)

### Task 4: Harden CSP for production
Edit `apps/frontend/next.config.ts`:

Current problematic line in `connect-src`:
```
"http://local-backend http://localhost:*"
```

This should only be present in development mode. Restructure:
```typescript
const connectSrcParts = [
  "'self'",
  "https://api.carbonscope.simu.space",
  "https://carbonscope-backend-app.azurewebsites.net",
  "https://*.supabase.co",
  "wss://*.supabase.co",
  "https://www.googletagmanager.com",
  "https://eu.i.posthog.com",
  "https://eu.posthog.com",
  "https://cloud.langfuse.com",
  "https://js.stripe.com",
];

// Only allow localhost in development
if (process.env.NODE_ENV === 'development') {
  connectSrcParts.push("http://local-backend", "http://localhost:*");
}
```

Then use `connectSrcParts.join(' ')` in the CSP string.

### Task 5: Full secret scan
```bash
# Scan for common API key patterns in tracked files
git ls-files | xargs grep -E '(sk-[a-zA-Z0-9]{20,}|ghp_[a-zA-Z0-9]{36}|AKIA[A-Z0-9]{16}|-----BEGIN.*PRIVATE KEY)' 2>/dev/null | grep -v node_modules | grep -v .next
```
If any found, remove from git and rotate immediately.

### Task 6: Commit
```bash
git add -A
git commit -m "security: remove .env.production from git, harden CSP

- Remove .env.production from git tracking (kept locally)
- Add .env.production to .gitignore
- Tighten CSP connect-src (localhost only in dev mode)
- Add .env.*.local to .gitignore"
```
